# My Constellation ✦

> 日々を照らす、小さな星たちを集めて。

推し・ライブ・映画・新曲・誕生日・試合予定など、自分を幸せにしてくれる予定だけを集める、
個人用の予定表 PWA（Progressive Web App）です。予定ひとつひとつを「星」として、
自分だけの星座を描いていくというコンセプトのアプリです。

すべてのデータは**この端末のブラウザ内（localStorage）**に保存されます。サーバーには一切送信されません。

---

## 特徴

- 📅 月表示カレンダー（土曜=青、日曜・祝日=赤で自動表示。日本の祝日・振替休日に対応）
- ✦ 「今日の予定」「これから1週間の予定」を2分割で表示
- 🔎 予定のタイトル・メモで検索、日付順／カテゴリー順に並び替え
- 🎨 カテゴリーごとに色を自由にカスタマイズ、新規カテゴリーも追加可能
- 🌗 夜空テーマ／やわらかテーマ（ライト）を切り替え可能
- 💾 JSONファイルへのバックアップ書き出し／復元（機種変更時も安心）
- 📱 ホーム画面に追加してアプリのように起動可能（オフラインでも利用可能）
- ✨ 誕生日カテゴリーは自動で「毎年繰り返す」がオンに
- 🫧 長押しで予定を編集、削除前には必ず確認ダイアログを表示

## フォルダ構成

```
/
├── index.html            アプリ本体（PWAのエントリーポイント）
├── manifest.json         PWAマニフェスト（アプリ名・アイコン・テーマカラーなど）

├── favicon.ico
├── icon-source.svg       アイコンの元データ（SVG）
├── icons/                各サイズのアプリアイコン・スプラッシュ画像
├── css/
│   └── style.css         スタイル一式（ダーク／ライトテーマ対応）
├── js/
│   ├── utils.js          日付計算・文字列などの共通処理
│   ├── holidays.js       日本の祝日・振替休日の計算
│   ├── storage.js        localStorage の読み書き、JSONバックアップ

│   └── app.js             状態管理・イベント処理・初期化（本体の司令塔）
└── README.md              このファイル
```

## 公開方法（GitHub Pages）

1. GitHubで新しいリポジトリを作成します（例：`my-constellation`）。
2. このフォルダの中身（`index.html` や `manifest.json` など）を**すべて**リポジトリのルート直下にアップロードします。
   - フォルダごとドラッグ&ドロップでアップロードすればOKです。
3. リポジトリの **Settings → Pages** を開きます。
4. 「Build and deployment」の Source を **Deploy from a branch** に設定し、
   Branch を `main`（または既定のブランチ）、フォルダを `/ (root)` に設定して **Save** します。
5. 数分待つと、`https://ユーザー名.github.io/リポジトリ名/` という形のURLでアプリが公開されます。
6. そのURLをスマホのSafari／Chromeで開いてください。

> 💡 `index.html` はJavaScriptモジュール（`type="module"`）を使っているため、
> ローカルでファイルを直接ダブルクリックして開く（`file://`）ことはできません。
> 必ずGitHub Pagesなど、`https://` で始まる場所から開いてください。

### ローカルで動作確認したいとき

Node.jsが入っていれば、プロジェクトフォルダ内で以下を実行すると簡易サーバーが立ちます。

```
npx serve .
```

表示されたURL（例：`http://localhost:3000`）をブラウザで開いて確認できます。

## 更新方法

1. ファイルを編集します（`index.html` / `css/style.css` / `js/*.js` など）。
2. 何かキャッシュ対象のファイルを変更したら、`service-worker.js` 内の

   ```js
   const CACHE_VERSION = "my-constellation-v1";
   ```

   の数字部分（`v1` → `v2` など）を必ず1つ上げてください。これを忘れると、
   スマホ側が古いキャッシュを使い続けてしまい、更新が反映されないことがあります。
3. 変更したファイルをGitHubリポジトリに再アップロード（コミット）します。
4. GitHub Pagesは自動で再デプロイされます（数分かかることがあります）。
5. スマホ側は、アプリを一度完全に閉じてから開き直すと新しいバージョンが反映されます。
   反映されない場合は、ホーム画面のアイコンを長押しして削除し、再度「ホーム画面に追加」し直してください。

## ホーム画面への追加方法（PWAとしてインストール）

### iPhone（Safari）
1. GitHub PagesのURLをSafariで開きます。
2. 画面下の共有ボタン（□に↑のアイコン）をタップします。
3. 「ホーム画面に追加」をタップします。
4. 名前を確認して「追加」をタップすると、ホーム画面にアイコンが追加されます。
5. 追加したアイコンから起動すると、Safariのアドレスバーなどが表示されない
   アプリのような画面（standalone表示）で開きます。

### Android（Chrome）
1. GitHub PagesのURLをChromeで開きます。
2. 右上の「⋮」メニューを開きます。
3. 「アプリをインストール」または「ホーム画面に追加」をタップします。
4. 案内に従って追加すると、ホーム画面からアプリのように起動できます。

## データのバックアップについて

予定とカテゴリーはブラウザの localStorage に保存されているため、**ブラウザのデータを消去したり
機種変更をするとデータが失われます**。アプリ内の「⚙️ 設定 → バックアップ」から、

- **JSONで書き出す**：現在のすべての予定・カテゴリーを1つのJSONファイルとしてダウンロード
- **JSONから復元**：書き出しておいたJSONファイルを選んで復元

ができます。機種変更の前や、定期的なタイミングでの書き出しをおすすめします。

## 技術メモ

- フレームワーク不要のバニラJavaScript（ESモジュール）で構成しています。
- 状態はメモリ上の単純なオブジェクトで管理し、変更のたびに `#app` の中身を再構築する
  シンプルな設計です（React等は使用していません）。
- 祝日判定は計算式によるもの（1980〜2099年で有効な標準的なルール）で、
  法改正等による例外年には対応していない場合があります。
- オフライン対応は Service Worker によるアプリシェルのキャッシュです。
  データの保存自体は常に localStorage（端末内）です。
